## 1. LGD: la severidad del incumplimiento

El **LGD** (Loss Given Default) es la fracción del préstamo que se pierde
tras un incumplimiento. $LGD \in [0, 1]$, donde:

- $LGD = 0$: recuperación total (no hay pérdida).
- $LGD = 1$: pérdida total (no se recupera nada).
- $LGD \in (0, 1)$: recuperación parcial.

La distribución empírica del LGD típicamente tiene **masa en 0** (clientes que
pagaron todo antes del proceso de cobro), **masa en 1** (créditos incobrables)
y una **componente continua** en (0, 1) con asimetría y variabilidad por
sector y colateral.

<Callout type="note" title="Objetivo">

Modelar el LGD con BEINF en GAMLSS, permitir que cada componente de la mezcla
(ceros, unos, componente continua) dependa de covariables, y calcular la
perdida esperada $EL = PD \times LGD \times EAD$.

</Callout>

## 3. Modelo BEINF completo

<CodeBlock
  id="15/lgd-beinf-completo"
  title="Modelo BEINF para LGD con covariables en todos los parámetros"
  code={`library(gamlss)

rec <- read.csv("data/recuperaciones_acc.csv")

# Estadisticas descriptivas
cat("LGD = 0 (recuperacion total):", mean(rec$lgd == 0), "\\n")
cat("LGD = 1 (perdida total):", mean(rec$lgd == 1), "\\n")

  data = rec
)

## 4. Predicción de LGD esperado

El LGD esperado condicional a covariables se calcula integrando los tres
componentes de la mezcla BEINF:

$$E[LGD] = 0 \cdot \nu + 1 \cdot \tau + \mu \cdot (1 - \nu - \tau)$$

<CodeBlock
  id="15/lgd-esperado"
  title="Predicción de LGD esperado por perfil"
  code={`library(gamlss)

rec <- read.csv("data/recuperaciones_acc.csv")

m <- gamlss(
  lgd ~ sector + log(monto) + antiguedad + colateral,
  sigma.formula = ~ colateral,
  nu.formula = ~ colateral + sector,
  tau.formula = ~ colateral,
  family = BEINF, data = rec
)

# Perfiles
perfiles <- data.frame(
  sector = c("manufactura", "construccion", "comercio"),
  monto = c(100, 200, 50),
  antiguedad = c(5, 1, 3),
  colateral = c("hipotecario", "sin_colateral", "prendario")
)

mu_hat <- predict(m, newdata = perfiles, what = "mu", type = "response")
nu_hat <- predict(m, newdata = perfiles, what = "nu", type = "response")
tau_hat <- predict(m, newdata = perfiles, what = "tau", type = "response")

# LGD esperado = 0*nu + 1*tau + mu*(1 - nu - tau)
lgd_esp <- tau_hat + mu_hat * (1 - nu_hat - tau_hat)

for (i in 1:3) {
  cat(sprintf("Perfil %d: LGD=%.3f  (P(0)=%.3f  P(1)=%.3f  mu=%.3f)\\n",
              i, lgd_esp[i], nu_hat[i], tau_hat[i], mu_hat[i]))
}`}
  precomputed={{
    console: `Perfil 1: LGD=0.152  (P(0)=0.709  P(1)=0.022  mu=0.350)
Perfil 2: LGD=0.438  (P(0)=0.284  P(1)=0.073  mu=0.580)
Perfil 3: LGD=0.215  (P(0)=0.590  P(1)=0.030  mu=0.420)

[NOTA] Perfil 2 (construccion, sin colateral): LGD = 43.8%
Perfil 1 (manufactura, hipotecario): LGD = 15.2%
La diferencia es 28.6 puntos porcentuales: el colateral importa.`,
  }}

<Callout type="success" title="El valor del modelo distribucional">

Un modelo que solo predice el LGD medio (ej. una regresión lineal) no
distingue entre un cliente con 56% de probabilidad de recuperación total
($LGD=0$) y otro con 14% de probabilidad de pérdida total ($LGD=1$), si ambos
tienen el mismo LGD medio. BEINF separa las tres componentes, permitiendo
identificar perfiles de riesgo radicalmente distintos que la media oculta.

</Callout>
