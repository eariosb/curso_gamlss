## 1. IBNR: los siniestros que aún no conocemos

En seguros de caución, el **IBNR** (Incurred But Not Reported) representa los
siniestros que ya ocurrieron pero que la aseguradora aún no ha registrado.
La reserva IBNR es el capital que ACC debe constituir para hacer frente a
estos siniestros pendientes.

El método clásico, **Chain Ladder**, estima las reservas usando un triángulo
de desarrollo: pagos acumulados por año de origen y año de desarrollo. Pero
Chain Ladder asume que el patrón de desarrollo es estable y no modela la
**incertidumbre** de las reservas. GAMLSS permite:

1. Modelar la **varianza** de las reservas (no solo el punto medio).
2. Incorporar **sobredispersión** creciente en los años de desarrollo tardíos.
3. Producir **intervalos de confianza** para las reservas.

Modelar reservas IBNR con GAMLSS usando el triángulo de desarrollo de ACC,
incorporar sobredispersión con NBI, y comparar las estimaciones con Chain

</Callout>

## 4. Modelo GAMLSS para el triángulo

El enfoque GAMLSS modela los **pagos incrementales** directamente:

<CodeBlock
  id="16/gamlss-ibnr"
  title="Modelo GAMLSS para reservas IBNR"
  code={`library(gamlss)

triangulo <- read.csv("data/triangulo_caucion.csv")

# Variables: año de origen y de desarrollo como factores
triangulo$origen_f <- factor(triangulo$origen)
triangulo$desarrollo_f <- factor(triangulo$desarrollo)

# Modelo NBI: media depende de origen y desarrollo
# sigma depende de desarrollo (sobredispersion creciente)
m <- gamlss(
  incremental ~ origen_f + desarrollo_f,
  sigma.formula = ~ desarrollo_f,
  family = NBI,
  data = triangulo
)

summary(m)

# Predecir pagos para las celdas del triangulo inferior (no observadas)
# (codigo simplificado: en la practica se generan las celdas faltantes)
pred <- predict(m, newdata = triangulo, type = "response")
cat("Reserva IBNR (GAMLSS):", round(sum(pred[is.na(triangulo$incremental)])),
    "millones COP\\n")

# Intervalo de confianza del 95%
sigma_hat <- predict(m, newdata = triangulo, what = "sigma", type = "response")
# Usar qNBI para el cuantil 97.5%
cat("Reserva IBNR P97.5%:", round(sum(qNBI(0.975,
    mu = pred, sigma = sigma_hat)[is.na(triangulo$incremental)])),
    "millones COP\\n")`}
  precomputed={{

<Callout type="success" title="La ventaja de GAMLSS para IBNR">

GAMLSS no solo estima la reserva central (172 vs 168 millones de Chain
Ladder), sino que produce **intervalos de confianza** naturales: P97.5% =
215 millones. Esto es crítico para el **capital de solvencia**: el regulador
exige reservar no solo el valor esperado, sino un cuantil alto que cubra
escenarios adversos. Chain Ladder requiere bootstrap o aproximaciones
adicionales para obtener intervalos; GAMLSS los produce directamente.

</Callout>

## 5. Sobredispersión creciente: la cola del desarrollo

El modelo muestra que $\sigma$ aumenta con el año de desarrollo: los pagos
tardíos son más inciertos. Esto refleja la realidad de los seguros de caución:
los siniestros de los primeros años de desarrollo son predecibles (la mayoría
ya se conocen), pero los de los años tardíos son **inciertos** (siniestros
largos, disputas legales, recuperaciones tardías).

<Callout type="warning" title="Modelar σ por desarrollo es esencial">

Si se asume $\sigma$ constante (como hace un GLM Poisson), se subestima la
incertidumbre de los pagos tardíos y se producen intervalos de confianza
demasiado estrechos. GAMLSS con `sigma.formula = ~ desarrollo_f` captura
esta heterocedasticidad de forma natural.

</Callout>
